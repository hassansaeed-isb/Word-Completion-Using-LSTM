Shakespeare Word Completion - Instructions

Setup Instructions

1. Train the model on Kaggle

Start by training the LSTM model on Kaggle with GPU acceleration:

  1. Go to Kaggle and create a new notebook
  2. Add the Shakespeare plays dataset to your notebook
  3. Copy the training code from the file kaggle_training.py into your notebook
  4. Run the notebook with GPU acceleration enabled (Settings → Accelerator → GPU)
  5. Wait for the training to complete (approximately 15-30 minutes depending on the GPU)
2. Download the model files

After training is complete, download the following files from the Output tab:

  • shakespeare_lstm_model.keras (or shakespeare_lstm_model.h5 for older versions)
  • tokenizer.pickle
  • model_params.txt (optional, but helpful)
3. Set up the Flask application

Now, set up the Flask application on your local machine:

  1. Create a new directory for the project
  2. Copy the Flask app code (app.py) into this directory
  3. Create a templates directory and add the HTML template files
  4. Place the downloaded model files in the same directory as app.py
4. Install required dependencies
pip install flask tensorflow numpy nltk pandas
5. Run the Flask application
python app.py

The application will be available at http://127.0.0.1:5000

Using the Application

Word Prediction

As you type text in the input area, the application automatically suggests the next word:

  1. Type a partial sentence in the text area
  2. The model will suggest the most likely next words
  3. Click on any suggestion to add it to your text
  4. Continue typing or clicking suggestions to build your sentence
Sentence Completion

You can also let the model automatically complete a sentence:

  1. Type a seed phrase or partial sentence
  2. Click the "Complete Sentence" button
  3. The model will generate additional words until it forms a complete sentence
Adjusting Settings

The application allows you to customize the model behavior:

  • Temperature: Controls the randomness of predictions. Higher values (>1.0) make the output more random, while lower values (<1.0) make it more deterministic.
  • Top-K: Controls how many word suggestions to display.
  • Max Words: Sets the maximum number of words to generate when completing a sentence.

Code Structure

The application consists of three main components:

  1. Model Training (on Kaggle):
    • Data preprocessing and tokenization
    • LSTM model creation and training
    • Model evaluation and saving
  2. Flask Backend:
    • Model loading and prediction
    • API endpoints for word prediction and sentence completion
    • Temperature-controlled text generation
  3. Web Interface:
    • Interactive text input area
    • Dynamic word suggestion display
    • Settings controls for customization
    • History of generated texts
